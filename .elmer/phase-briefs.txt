Phase 0a: Prove — Stakeholder Brief. Single-book semantic search proof with Autobiography of a Yogi. Covers: ingestion pipeline, hybrid search API, search UI, book reader, quality evaluation. Key SRF decisions: edition confirmation, PDF source, search quality threshold.
---
Phase 0b: Foundation — Stakeholder Brief. Deploy to Vercel, add AI librarian intelligence (query expansion, intent classification, passage ranking), build homepage with Today's Wisdom, establish observability, crisis query detection. Key SRF decisions: cultural design consultation, editorial voice guidelines.
---
Phase 1: Build — Stakeholder Brief. Build every page of the portal, establish SRF visual identity, deliver complete navigation experience. Covers: homepage, library, Quiet Corner, about page, sharing, SEO, accessibility, i18n infrastructure, testing pipeline, Terraform. Key SRF decisions: visual identity approval, newcomer pathways.
---
Phase 2: Read — Stakeholder Brief. Contemplative reader interactions: dwell mode, keyboard navigation, lotus bookmarks, reader typography, offline support, presentation mode for group reading. Key SRF decisions: contemplative interaction patterns, group reading use cases.
---
Phase 3: Grow — Stakeholder Brief. Multi-book ingestion to reach critical mass. Four waves of books from most quotable to most complex. Cross-book search, expanded Today's Wisdom and Quiet Corner pools. Key SRF decisions: book ingestion priority, content review process per book.
---
Phase 4: Operate — Stakeholder Brief. Transform portal from technical project to running content operation. Editorial review portal, theme navigation (Doors of Entry), Living Glossary, calendar-aware content, staff tools, The Spiritual Guide pathways. Key SRF decisions: editorial staffing (coordinator, editor, theological reviewer), theme taxonomy review, feature flag activation timing.
---
Phase 5: Connect — Stakeholder Brief. Related Teachings system — the feature that makes the portal irreplaceable. Pre-computed cross-book connections, reader side panel, editorial reading threads, reverse bibliography, People Library. Key SRF decisions: relation quality standards, editorial thread curation approach.
---
Phase 6: Complete — Stakeholder Brief. Complete the full Yogananda library. Ingest all remaining book waves including scriptural commentaries. Add production observability, search analytics, "What Is Humanity Seeking?" public dashboard, knowledge graph visualization. Key SRF decisions: verse-aware chunking approach for commentaries, public dashboard editorial policy.
---
Phase 7: Empower — Stakeholder Brief. Tools for monastics, center leaders, and study circles. PDF downloads, study guides, Study Workspace for talk preparation, study circle sharing, magazine integration. Key SRF decisions: magazine content scope, study circle sharing guidelines, monastic workflow needs.
---
Phase 8: Distribute — Stakeholder Brief. Extend reach beyond direct visitors. Daily email delivery, social media asset generation, WhatsApp integration, RSS feeds, calendar reading journeys, external MCP server for third-party access, events section, Sacred Places. Key SRF decisions: email editorial voice, social media approval workflow, WhatsApp scope, third-party MCP access policy.
---
Phase 9: Integrate — Stakeholder Brief. Migrate to Contentful CMS as editorial source of truth and GitLab as SRF's identity provider. Regional distribution via Neon read replicas. Key SRF decisions: Contentful contract, GitLab migration timing, regional deployment priority.
---
Phase 10: Translate — Stakeholder Brief. Serve portal in 9+ languages. Two parallel waves: Western (es, de, fr, it, pt, ja) and Indian (hi, bn). Per-language search, translation review UI, English fallback policy. Key SRF decisions: launch language priority, translation reviewer identification, YSS coordination for Indian languages.
---
Phase 11: Polish — Stakeholder Brief. Elevate UI to digital sanctuary quality. Formal WCAG audit, Calm Technology design system, PWA for offline reading, contemplative features (circadian color, opening moment, breath transitions). Key SRF decisions: third-party accessibility auditor, design system scope, ambient audio policy.
---
Phase 12: Multimedia — Stakeholder Brief. Cross-media intelligence. Video catalog with transcript search, audio library, sacred image handling, cross-media search, unified content hub. Key SRF decisions: video content authority hierarchy, sacred image handling protocol, chant inclusion scope.
---
Phase 13: Personalize — Stakeholder Brief. Optional user accounts for personal study features. Anonymous access remains default. Cross-device bookmark sync, reading progress, personalized daily passages. Key SRF decisions: whether to offer accounts at all, authentication method, DELTA compliance for authenticated users.
---
Phase 14: Community — Stakeholder Brief. Bridge solitary study with global SRF community. Event calendar, local center discovery, SMS access for basic phones, Telegram bot, Community Collections gallery, Voluntary League of Devotees curation program. Key SRF decisions: VLD program structure, SMS regional partners, community moderation policy, annual narrative report editorial process.
